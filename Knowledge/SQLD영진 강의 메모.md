엔터티

- 저장하고자 하는 대상

- 업무에 필요할 것

- 집합적(인스턴스 2개 이상)

- 포괄적(너무 세세하지 않게)

- 속성을 1개 이상 가질것

- 다른 엔터티와 1개 이상 관게를 가질것

- 식별자가 존재할 것

# 속성

## 속성의 특징

업무상 필요호 하는 데이터

의미상 원자값

엔티티의 공통적인 특징을 설명하고 인스턴스의 구체적 내용 설명

주식별자에 함수적 종속됨

하나의  속성은 하나의 속성값만 가진다

## 속성의 종류

특성에 따른 분류

- 기본속성: 요구사항로부터 직접 도출한 속성

- 설계속성: 편의를 위해 인위적으로 만든 속성(상품ID값 등)

- 파생속성: 다른 속성들로부터 계산이 되는 속성

엔터니 구성 방식에 따른 분류

- PK

- FK
  - 식별자나 유니크키만 빌려줄 수 있음

- 일반 속성

## DB 종류

1. 계층형
  - 레지스트리 등
2. 네트워크형
3. 관계형

## 관계의 종류

1. 존재적 관계(연관)
  - 단순히 소속되거나 포함되는 관계
2. 행위적 관계(의존)
  - 행위로 인해 연결되는 관계

ERD: 데이터 모델링
- 존재적 관계: 실선
- 행위적 관계: 실선
UML: 프로세스 모델링
- 존재적 관계: 실선
- 행위적 관계: 점선

## ERD

관계 표현 3요소
- 관계명(membership): 두 엔터티간 논리적인 관계
  - 현재동사
  - 양방향이라 두번써야할수도
  - IE, 바커 표ㅣㄱ법 동일
- 관계차수(degree/cardinality): 인스턴스끼리 얼마나 관계에 참여하는지(1:1, 1:N, N:M 등)
  - IE표기법에선 1 작대기 쓰고 바커는 안씀
- 관계선택성(optionality): 인스턴스가 관게에 반드시 참여해야 하는지
  - IE: O 있으면 필수 아니고 선택(optional)
  - 바커: 반대쪽 선을 점선으로 표시

관계 체크사항
- 두 엔터티간 관심있는 연관 규칙이 있는가?
- 두 엔터티 사이에 정보의 조합이 발생하는가?
- 업무기술서, 장표에 관계연결에 대한 규칙이 서술되어있는가?(관계차수, 선택성)
- 업무기술서, 장표에 관계연결을 가능케 하는 동사가 있는가?(관계명)

ERD: 엔터티간의 논리적인 관계를 그린 그림

필요성

- 복잡한 요구사항 시각화
- 구성원간 의사소통 도구
- 개념적 모델링 산출물

ERD 그리는 순서
1. 엔터티 도출
2. 엔터티 배치: 중심이 되는 엔터티를 좌상단에 배치
3. 관계선 그리기
4. 관계명 작성
5. 관계차수 작성
6. 관계 선택성 표시

코드성 엔터티와 통계성 엔터티는 다른 엔터티들과 관계가 없어도 된다

# 식별자

엔터티 안의 인스턴스를 유일하도록 식별할 수 있는 속성의 집합

모든 엔터티는 하나 이상의 식별자를 가져야 함

식별자의 분류
- 대표성 여부: 엔터티를 대표하는가
  - 주식별자(PK): 보조식별자 중 하나
    - 특징
      - 유일성: 인스턴스를 유일하게 식별
      - 최소성: 최소한의 속성 조합으로 구성
      - 불변성: 잘 안변해야함
      - 존재성: 존재해야만 함. null안됨
    - 도출 기준
      - 업무에서 자주 사용될 것
      - 이름, 내역 같이 이름으로 되거나 불명확한것은 배제
      - 너무 많은 속성으로 이뤄지지 X
  - 보조 식별자: 선택되지 않은 나머지
- 스스로 발생 여부: 엔터티에 원래 있던건가
  - 내부 식별자(주로 PK)
  - 외부 식별자(FK)
- 단일 속성 여부: 속성 하나로 구성되었는가
  - 단일 식별자
  - 복합 식별자
- 대체 여부: 업무상 편의를 위해 대체한 것인가(복합이던 아니건 무관)
  - 본질식별자
  - 인조식별자


식별 관계/비식별 관계


식별관계(강한연결관계): FK 가 주식별자의 일부
- IE 표기법: 실선
- 바커 표기법: 1로 표시
- 값 필수
- 데이터 독립 생성 불가능, 부모 필수
- 부모 삭제시 자식도 삭제
- 상속할때마다 받는놈은 주식별자 개수 늘어남(확장)
비식별 관계(약한연결관계): FK가 일반속성
- IE 표기법: 점선
- 바커 표기법: 표시 없음
- 값 선택
- 데이터 독립 생성 가능, 부모 필수 아님
- 부모 삭제시 자식은 남음
- 상속해도 주식별자 개수 유지(차단)

식별만 쓰면 식별자가 너무 많아져 조건이 많아짐

비식별만 쓰면 조인이 너무 증가함
